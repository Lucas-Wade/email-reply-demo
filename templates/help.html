系统概览

本系统自动读取企业邮箱中的新邮件，通过 AI 三层分类判断是否为有效询盘，并自动生成回复草稿，等待人工审核后一键发送。

系统默认每 5 分钟自动检查一次邮件，也可点击导航栏"检查邮件"按钮立即触发。
首次运行
  1. 复制 .env.example 为 .env，填入邮箱账号、API Key 等配置
  2. 设置 ADMIN_EMAIL 和 ADMIN_PASSWORD（首次启动自动创建管理员账号）
  3. 运行命令启动服务： 
    venv/Scripts/python -m uvicorn main:app --port 8000
  4. 浏览器访问 http://localhost:8000，用管理员邮箱和密码登录
  5. 登录后立即前往右上角用户菜单 → 修改密码，更换初始密码
  6. 点击导航栏"检查邮件"测试邮箱连接是否正常
询盘处理流程
三层分类标准
层级方式判定结果
Layer 0黑白名单规则黑名单 → 垃圾；白名单 → 直接生成草稿
Layer 1Python 规则（无 API）明显垃圾邮件直接过滤，节省 API 调用
Layer 2LLM 8 维度评分≥4 分 = 有效询盘；2-3 分 = 其他；<2 分 = 垃圾
主列表操作
  • 点击任意行 → 展开查看邮件正文、分类评分明细、意向强度
  • 点击"查看草稿"按钮进入草稿审核页
  • 邮件展开时自动标记为已读
  • 状态说明：新邮件 → 待审核 → 已发送
草稿审核
页面左侧
  • 原始询盘全文
  • 发件人、时间、语言
  • 意向强度（1-5 星）
  • 关联原始询盘（Re: 邮件时显示）
页面右侧
  • AI 匹配的产品报价（可有多个）
  • 可直接编辑的回复草稿
  • 批准发送 / 拒绝 两个操作
建议：修改完草稿先点"保存修改"，再点"批准并发送"，防止内容丢失。 批准后系统自动创建 3 天后的跟进提醒。
跟进提醒

报价发送后，系统自动在 3 天后创建跟进任务。每天 09:00 自动检查是否有逾期任务。

状态含义
待处理到期日未到
已逾期超过到期日未处理
草稿已生成AI 已生成跟进草稿，待确认发送
已发送跟进邮件已发出
已跳过手动标记为不跟进

点击"生成草稿"→ AI 自动写跟进邮件 → 检查内容 → 点"发送"

客户档案

系统从每封有效询盘的发件人域名自动建档，记录：

  • 公司名称 / 国家
  • 买家类型（终端用户/贸易商/经销商/代理商）
  • 询盘次数 / 已报价次数 / 跟进次数
  • 首次 / 最近联系时间

点击"历史"查看该客户所有往来邮件。无需手动维护，系统自动更新。

黑白名单（规则管理）
黑名单

匹配的发件人直接标为垃圾，不调用 AI，不显示草稿。
适合：已知竞争对手、广告公司、骚扰发件人

白名单

匹配的发件人跳过 AI 分类，直接生成草稿。
适合：老客户、集团内部地址、认证合作商

匹配模式支持域名（如 spam.com）或完整邮件地址，大小写不敏感，有任意包含即触发。

数据分析
  • 询盘漏斗：有效询盘 → 已草稿 → 已发送 → 已跟进，各环节转化率
  • 邮件分类分布：有效询盘 / 垃圾 / 其他占比饼图
  • 近 8 周趋势：有效询盘数量周度柱状图
  • 客户国家分布：Top 10 来源国排行
  • 关键指标：平均响应时长（小时）、草稿→发送转化率
背景调查

在草稿审核页点击"背景调查"，系统将通过 Google 搜索对发件人公司进行三轮查询：

  1. 公司基本信息（成立时间、规模、官网）
  2. 域名信誉（是否有诈骗投诉）
  3. 行业匹配度（是否真实采购硬质合金产品）

结果包括：风险等级（低/中/高）、买家类型、红旗警告、加分项、综合建议。

需要在 .env 中配置 SERPER_API_KEY 才能使用。
